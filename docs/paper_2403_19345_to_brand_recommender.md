# 2403.19345 논문을 브랜드 영업 추천 자동화에 적용하는 방식

## 논문의 핵심

논문 `Intelligent Classification and Personalized Recommendation of E-commerce Products Based on Machine Learning`은 이커머스 상품 추천을 다음 흐름으로 설명한다.

1. 상품명, 설명, 카테고리 같은 상품 텍스트를 모은다.
2. 결측값과 긴 텍스트를 전처리한다.
3. BERT로 상품 텍스트를 벡터화한다.
4. 사용자의 구매 이력과 가까운 상품을 nearest neighbor로 찾는다.
5. 고객별 추천 상품 리스트를 만든다.

논문이 강조하는 문제는 네 가지다.

- 데이터 프라이버시
- 알고리즘 편향
- 확장성
- 콜드 스타트

네 프로젝트에서는 개인정보 기반 개인화가 아니라 브랜드 공개 리뷰/FAQ 기반 B2B 추천이므로, 개인정보 위험은 상대적으로 낮다. 대신 더 중요한 문제는 노이즈, 오분류, 근거 부족이다.

## 네 데이터에 맞춘 변환

논문의 `상품`은 여기서 `브랜드`가 된다.

| 논문 | 네 프로젝트 |
| --- | --- |
| item title | 브랜드명, 대표상품, 현재 밀고 있는 상품 |
| item description | 리뷰, FAQ, 장점, 단점, 가격 신호 |
| BERT item vector | 브랜드 의미 벡터 |
| nearest product | 유사 브랜드 |
| next purchase prediction | 다음에 제안할 영업상품 추천 |
| cold start product | 리뷰가 적은 신규/약근거 브랜드 |

즉 목표는 상품 추천이 아니라 다음 문장이다.

> 브랜드의 리뷰/FAQ/장단점 텍스트를 기반으로 비슷한 브랜드를 찾고, 그 브랜드들에서 반복되는 영업상품을 추천한다.

## 지금 구현한 로컬 버전

파일: `build_brand_semantic_recommender.py`

지금은 API와 외부 모델 다운로드 없이 안정적으로 돌리기 위해 BERT 대신 로컬 TF-IDF 벡터를 사용한다.

- 입력: `ttamtti2_local_review_analysis.csv`
- 브랜드 문서: 브랜드명 + 대표상품 + 그룹 + 키워드 + 리뷰/FAQ/장단점 + 기존 추천상품
- 벡터화: 단어 토큰 + 한국어 2~4글자 n-gram TF-IDF
- 추천: cosine similarity + 같은 그룹/세부유형 보정 + 근거등급 보정
- 출력: 브랜드별 유사 브랜드, 의미 기반 추천 영업상품, 추천 근거

이 구조는 추후 BERT로 바꾸기 쉽다. `build_brand_document()`로 브랜드 문서를 만든 뒤, `build_tfidf()` 부분만 sentence-transformer 또는 KoBERT 임베딩으로 교체하면 된다.

## 왜 이 방식이 네 일에 맞는가

현재 규칙 기반 그룹 분류는 `보습`, `진정`, `트러블`, `배송`, `가격` 같은 명확한 키워드에는 강하다. 하지만 브랜드마다 표현이 다르면 같은 의미인데도 다르게 잡힐 수 있다.

의미 유사도 추천을 붙이면 다음이 가능해진다.

- 같은 키워드를 쓰지 않아도 비슷한 리뷰 맥락의 브랜드를 묶는다.
- 한 브랜드의 추천상품을 비슷한 브랜드에 전이할 수 있다.
- 리뷰가 적은 브랜드도 브랜드명/대표상품/FAQ만으로 가까운 그룹을 찾을 수 있다.
- PPT와 제안메일에서 "비슷한 브랜드 근거"를 추가할 수 있다.

## 실무 사용 방식

1. 규칙 기반 분석으로 1차 그룹과 근거등급을 만든다.
2. 의미 유사도 추천으로 가까운 브랜드를 찾는다.
3. 유사 브랜드들의 추천 영업상품을 투표시킨다.
4. 기존 추천상품과 의미 기반 추천상품이 같으면 발송 우선순위를 올린다.
5. 서로 다르면 검토필요로 표시하고 사람이 확인한다.

## 주의점

이 논문은 아이디어 참고로는 좋지만, 성능 검증은 강한 편이 아니다. 특히 실제 수치 평가보다 수동 평가 설명이 많다. 그래서 네 시스템에서는 논문을 그대로 믿기보다, 다음 기준으로 자체 검증하는 것이 좋다.

- 유사 브랜드 3개가 사람이 보기에도 같은 상품군인가
- 추천 영업상품이 리뷰 근거와 맞는가
- 검토필요 브랜드가 너무 많이 자동 발송되지 않는가
- PPT에 들어갈 대표문장이 노이즈가 아닌가
